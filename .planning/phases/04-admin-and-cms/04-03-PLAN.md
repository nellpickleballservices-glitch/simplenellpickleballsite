Append to `app/actions/admin.ts`:

**Court Actions:**

1. `getCourtsAction()`:
   - requireAdmin(). Query supabaseAdmin.from('courts').select('*, locations(name, address)').order('name').
   - Return courts with joined location data.

2. `addCourtAction(formData: FormData)`:
   - requireAdmin(). Extract: locationName, courtName, address, lat, lng, capacity from formData.
   - First, find or create location: supabaseAdmin.from('locations').upsert({ name: locationName, address, lat: parseFloat(lat), lng: parseFloat(lng) }, { onConflict: 'name' }) -- if location with same name exists, reuse it.
   - Then insert court: supabaseAdmin.from('courts').insert({ location_id, name: courtName, status: 'open', lat: parseFloat(lat), lng: parseFloat(lng) }).
   - Also create default court_config (weekday + weekend) and court_pricing (full_court + open_play) entries matching existing seed data patterns.
   - Return { success: true }.

3. `setMaintenanceAction(courtId: string, startDate: string, endDate: string)`:
   - requireAdmin().
   - Update court: supabaseAdmin.from('courts').update({ status: 'maintenance', maintenance_start: startDate, maintenance_end: endDate }).eq('id', courtId).
   - Cancel overlapping reservations: supabaseAdmin.from('reservations').update({ status: 'cancelled' }).eq('court_id', courtId).gte('starts_at', startDate).lte('starts_at', endDate).in('status', ['confirmed', 'pending_payment']).select('user_id, reservation_user_first_name, starts_at').
   - Send cancellation notification emails to affected users via Resend (fire-and-forget, try/catch with console.error on failure).
   - Return { success: true, cancelledCount: affected?.length ?? 0 }.

4. `clearMaintenanceAction(courtId: string)`:
   - requireAdmin().
   - Update court: supabaseAdmin.from('courts').update({ status: 'open', maintenance_start: null, maintenance_end: null }).eq('id', courtId).
   - Return { success: true }.

**Reservation Actions:**

5. `getAllReservationsAction(filters: { dateFrom?: string; dateTo?: string; courtId?: string; status?: string; page?: number })`:
   - requireAdmin().
   - Build query on supabaseAdmin.from('reservations').select('*, courts(name)', { count: 'exact' }).
   - Apply filters: .gte('starts_at', dateFrom), .lte('starts_at', dateTo), .eq('court_id', courtId), .eq('status', status).
   - Pagination: 20 per page.
   - Also join profile data: separate query for user names by user_ids (or use reservation_user_first_name/last_name snapshots already on the row).
   - Order by starts_at desc.
   - Return { reservations, total, page }.

6. `adminCancelReservationAction(reservationId: string)`:
   - requireAdmin().
   - supabaseAdmin.from('reservations').update({ status: 'cancelled' }).eq('id', reservationId).
   - Return { success: true }.

7. `adminCreateReservationAction(formData: FormData)`:
   - requireAdmin(). Get admin user from requireAdmin return.
   - Extract: userId (optional - if registered user), guestName (optional - if walk-in), courtId, startsAt, endsAt, bookingMode, spotNumber.
   - If userId provided: create reservation with that user_id, snapshot their name from profiles.
   - If guestName provided (no userId): create reservation with admin's user_id, guest_name set, created_by_admin=true.
   - Set payment_status to 'cash_pending' for walk-in guests, 'free' for members.
   - Return { success: true }.

8. `markCashPaidAction(reservationId: string)`:
   - requireAdmin().
   - supabaseAdmin.from('reservations').update({ payment_status: 'paid', payment_method: 'cash' }).eq('id', reservationId).
   - Return { success: true }.

9. `searchUsersForReservationAction(query: string)`:
   - requireAdmin(). Quick search returning top 10 matching users (id, first_name, last_name, email) for the reservation creation dropdown.
   - Search profiles by name + auth.users by email. Return merged list.

Update i18n messages (append to Admin namespace in en.json + es.json):
- Courts section: addCourt, courtName, locationName, address, latitude, longitude, capacity, maintenanceMode, setMaintenance, clearMaintenance, startDate, endDate, maintenanceActive, courtOpen, courtClosed
- Reservations section: allReservations, filterByDate, filterByCourt, filterByStatus, cancelReservation, createReservation, onBehalf, selectUser, guestWalkIn, guestName, markCashPaid, cashPending, confirmed, cancelled, pendingPayment
