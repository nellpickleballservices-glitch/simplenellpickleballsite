  <action>
Add the following Server Actions to `app/actions/admin.ts` (append to existing file from Plan 01):

1. `searchUsersAction(query: string, page: number = 1)`:
   - Call requireAdmin().
   - Two-pronged search strategy:
     a. Search profiles table via supabaseAdmin: `.or('first_name.ilike.%${query}%,last_name.ilike.%${query}%,phone.ilike.%${query}%')` with pagination (pageSize=20, range offset).
     b. Search auth users for email: `supabaseAdmin.auth.admin.listUsers()` then client-side filter by email containing query (Supabase admin API lacks server-side email filter).
   - Merge results by user ID, deduplicate. For each user, join email from auth.users with profile data.
   - Return { users: UserWithDetails[], total: number, page: number }.
   - When query is empty, return paginated list of all users ordered by created_at desc.
   - Include membership status by left-joining memberships table.
   - Include is_banned by checking auth user's banned_until field.

2. `getUserDetailsAction(userId: string)`:
   - Call requireAdmin().
   - Fetch profile from supabaseAdmin.from('profiles').
   - Fetch auth user via supabaseAdmin.auth.admin.getUserById(userId) for email and ban status.
   - Fetch membership from supabaseAdmin.from('memberships').eq('user_id', userId).maybeSingle().
   - Fetch reservation history from supabaseAdmin.from('reservations').eq('user_id', userId).order('starts_at', { ascending: false }).limit(20).
   - Return combined object with profile, email, membership details, reservation history, is_banned.

3. `disableUserAction(userId: string)`:
   - Call requireAdmin().
   - supabaseAdmin.auth.admin.updateUserById(userId, { ban_duration: '876000h' }) -- ~100 years ban.
   - Auto-cancel future reservations: supabaseAdmin.from('reservations').update({ status: 'cancelled' }).eq('user_id', userId).gt('starts_at', new Date().toISOString()).in('status', ['confirmed', 'pending_payment']).
   - Return { success: true }.

4. `enableUserAction(userId: string)`:
   - Call requireAdmin().
   - supabaseAdmin.auth.admin.updateUserById(userId, { ban_duration: 'none' }).
   - Return { success: true }.

5. `triggerPasswordResetAction(userId: string)`:
   - Call requireAdmin().
   - Get user email: supabaseAdmin.auth.admin.getUserById(userId).
   - Generate recovery link: supabaseAdmin.auth.admin.generateLink({ type: 'recovery', email: user.email }).
   - Send via Resend (import from @/lib/resend) with bilingual template: "Your password reset link: {link}".
   - Return { success: true }.

Update i18n messages (en.json + es.json) - add to Admin namespace:

- membershipActive, membershipCancelled, membershipPastDue, noMembership
- reservationHistory, noReservations, viewDetails
- confirmDisableTitle, confirmDisableMessage, confirmEnableTitle, confirmEnableMessage
