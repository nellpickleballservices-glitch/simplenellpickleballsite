<task type="auto">
  <name>Task 1: Migration 0003 + Type Definitions</name>
  <files>supabase/migrations/0003_reservations.sql, lib/types/reservations.ts</files>
  <action>
Create `supabase/migrations/0003_reservations.sql` with the following in order:

1. Enable btree_gist extension: `CREATE EXTENSION IF NOT EXISTS btree_gist;`

2. Create `court_config` table (admin-managed schedule per court per day type):
   - id UUID PK, court_id UUID FK to courts, day_type TEXT ('weekday'|'weekend'), open_time TIME, close_time TIME, full_court_start TIME (nullable), full_court_end TIME (nullable), open_play_start TIME (nullable), open_play_end TIME (nullable)
   - UNIQUE(court_id, day_type)
   - RLS: all authenticated can SELECT, service_role full access

3. Create `app_config` table (key-value config):
   - key TEXT PK, value JSONB NOT NULL
   - RLS: all authenticated can SELECT, service_role full access
   - Seed default values: member_advance_booking_hours=72, non_member_advance_booking_hours=24, cancellation_window_hours=2, vip_guest_limit=1, pending_payment_hold_hours=2, session_price_default=10

4. Create `court_pricing` table:
   - id UUID PK, court_id UUID FK to courts, mode TEXT ('full_court'|'open_play'), price_cents INT DEFAULT 1000
   - UNIQUE(court_id, mode)
   - RLS: all authenticated can SELECT, service_role full access

5. ALTER TABLE reservations to add columns:
   - booking_mode TEXT NOT NULL DEFAULT 'open_play' ('full_court'|'open_play')
   - spot_number INT (1-4 for open play, NULL for full court)
   - payment_status TEXT DEFAULT 'free' ('free'|'pending_payment'|'paid'|'cash_pending')
   - payment_method TEXT (nullable: 'stripe'|'cash'|null)
   - stripe_payment_id TEXT (nullable)
   - guest_name TEXT (nullable, VIP guest full name)
   - price_cents INT DEFAULT 0

6. Add exclusion constraints (partial -- only for non-cancelled/non-expired):
   - `no_double_booking_open_play`: EXCLUDE USING GIST (court_id WITH =, spot_number WITH =, tstzrange(starts_at, ends_at) WITH &&) WHERE (status NOT IN ('cancelled', 'expired') AND booking_mode = 'open_play' AND spot_number IS NOT NULL)
   - `no_double_booking_full_court`: EXCLUDE USING GIST (court_id WITH =, tstzrange(starts_at, ends_at) WITH &&) WHERE (status NOT IN ('cancelled', 'expired') AND booking_mode = 'full_court')

7. Update RLS on reservations: add policy "All authenticated can read court reservations" FOR SELECT USING (true) -- needed for availability display. Drop or replace the owner-only SELECT policy since availability requires all users to see booking status (not personal details -- app layer filters).

8. Seed data: Insert 1 location ("NELL Pickleball Club", address, lat/lng for Santo Domingo area), 3 courts at that location with lat/lng, court_config for each court (weekday and weekend schedules), court_pricing for each court (full_court and open_play modes at $10).

Then create `lib/types/reservations.ts` with TypeScript types:
- `BookingMode = 'full_court' | 'open_play'`
- `PaymentStatus = 'free' | 'pending_payment' | 'paid' | 'cash_pending'`
- `ReservationStatus = 'confirmed' | 'pending_payment' | 'paid' | 'cancelled' | 'expired'`
- `Reservation` interface matching the enhanced table
- `CourtConfig` interface matching court_config table
- `CourtPricing` interface matching court_pricing table
- `AppConfigKey` union type for known config keys
- `TimeSlot` interface: { startTime: string, endTime: string, mode: BookingMode, spots: SpotInfo[] }
- `SpotInfo` interface: { spotNumber: number, isAvailable: boolean, reservedBy?: string }
- `CourtWithConfig` combining court + config + pricing for the UI

All types exported. Use `Database` naming convention consistent with Supabase.
