1. Create `supabase/functions/session-reminder/index.ts` (Deno Edge Function):

```typescript
// Supabase Edge Function: session-reminder
// Triggered every minute by pg_cron via pg_net
// Finds reservations ending in ~10 minutes, sends reminder email, sets reminder_sent=true
```

Flow:
- Create Supabase admin client using `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` from Deno.env
- Get RESEND_API_KEY from Deno.env
- Calculate time window: now + 10 minutes to now + 11 minutes (1-minute window matching cron frequency)
- Query reservations where:
  - reminder_sent = false
  - status IN ('confirmed', 'paid') -- not pending_payment, cancelled, or expired
  - ends_at >= tenMinLater AND ends_at < elevenMinLater
- Join with profiles for first_name and locale_pref
- Join with courts for court name
- For each matching reservation, get the user's email from auth.users (via supabaseAdmin.auth.admin.getUserById)
- Send bilingual reminder email via Resend REST API (fetch, not npm):
  - POST https://api.resend.com/emails
  - Authorization: Bearer {RESEND_API_KEY}
  - Body: { from, to, subject, text }
  - Bilingual: check profile.locale_pref for 'es' or 'en'
  - English: "Your pickleball session at {courtName} ends in 10 minutes. Please prepare to exit the court so the next group can begin."
  - Spanish: "Tu sesion de pickleball en {courtName} termina en 10 minutos. Por favor, preparate para salir de la cancha para que el siguiente grupo pueda comenzar."
  - NOTIF-03 exact text per requirements
- After sending, update reservation: SET reminder_sent = true (NOTIF-04)
- As a side effect: also clean up expired pending_payment holds (query pending_payment reservations where created_at + hold_window < NOW(), set status = 'expired'). Get hold window from app_config.
- Return Response('OK', { status: 200 })
- Wrap everything in try/catch; individual email failures should not stop processing other reservations

IMPORTANT Deno-specific notes:
- Use `Deno.serve()` handler pattern (NOT `export default`)
- Use `Deno.env.get('KEY')` not `process.env.KEY`
- Import Supabase client from `https://esm.sh/@supabase/supabase-js@2`
- Use native `fetch()` for Resend API calls (no npm imports)

2. Create `supabase/migrations/0004_pg_cron_reminder.sql`:

Enable pg_cron and pg_net extensions, then schedule the reminder:
```sql
-- Enable required extensions
CREATE EXTENSION IF NOT EXISTS pg_cron;
CREATE EXTENSION IF NOT EXISTS pg_net;

-- Store Edge Function URL and anon key in Vault for security
-- NOTE: These values must be set by the user in Supabase Dashboard > Vault
-- before the cron job will work. Placeholder SQL:
SELECT vault.create_secret('https://YOUR_PROJECT_REF.supabase.co', 'project_url');
SELECT vault.create_secret('YOUR_ANON_KEY', 'anon_key');

-- Schedule: every minute
SELECT cron.schedule(
  'session-reminder',
  '* * * * *',
  $$
  SELECT net.http_post(
    url := (SELECT decrypted_secret FROM vault.decrypted_secrets WHERE name = 'project_url')
           || '/functions/v1/session-reminder',
    headers := jsonb_build_object(
      'Content-Type', 'application/json',
      'Authorization', 'Bearer ' || (SELECT decrypted_secret FROM vault.decrypted_secrets WHERE name = 'anon_key')
    ),
    body := '{}'::jsonb
  ) AS request_id;
  $$
);
```

- RESEND_API_KEY must be set via `supabase secrets set RESEND_API_KEY=xxx`
