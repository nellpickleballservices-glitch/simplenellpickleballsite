Purpose: Eliminate the #1 performance bottleneck -- every page load currently triggers a getUser() roundtrip to Supabase Auth, plus a membership DB query on /member/* routes. After this change, public pages have zero auth overhead and member pages skip DB queries for 5 minutes.
Output: Rewritten middleware.ts with route-scoped auth and membership caching.

```typescript
const PROTECTED_PREFIXES = ['/member/', '/admin/', '/dashboard']
const AUTH_REDIRECT_ROUTES = ['/login', '/signup']
const RESERVATION_SEGMENTS = ['/reservations', '/checkout-session']

export function isProtectedRoute(pathname: string): boolean {
  return PROTECTED_PREFIXES.some(prefix => pathname.includes(prefix))
}

export function isAuthRedirectRoute(pathname: string): boolean {
  return AUTH_REDIRECT_ROUTES.some(route => pathname.endsWith(route))
}

export function needsAuthCheck(pathname: string): boolean {
  return isProtectedRoute(pathname) || isAuthRedirectRoute(pathname)
}

export function isReservationRoute(pathname: string): boolean {
  return RESERVATION_SEGMENTS.some(segment => pathname.includes(segment))
}
```

  <action>
Rewrite `middleware.ts` following the architecture from RESEARCH.md Pattern 2, using the route helpers created in Task 1 and the cookie signing utility from Plan 01.

**Structure:**

1. Import `intlMiddleware` from next-intl (keep existing pattern)
2. Import route helpers from `@/lib/middleware/route-helpers`
3. Import `setMembershipCookie`, `getMembershipFromCookie` from `@/lib/middleware/cookie-signing`

4. **Main middleware function logic:**

   a. Extract `pathname` from `request.nextUrl.pathname`

   b. **PUBLIC ROUTES** (`!needsAuthCheck(pathname)`): Return `intlMiddleware(request)` immediately. No Supabase client created. No getUser() call.

   c. **PROTECTED/AUTH ROUTES**: Create Supabase server client (same cookie pattern as current middleware). Call `supabase.auth.getUser()`.

   d. **Auth redirect routes** (`isAuthRedirectRoute(pathname)`):
      - If user IS logged in: redirect to `/member/dashboard`
      - If user is NOT logged in: return `intlMiddleware(request)` with Supabase cookies composed

   e. **Unauthenticated on protected route**: Redirect to `/login`

   f. **Admin routes** (`pathname.includes('/admin')`): Check `user.app_metadata?.role === 'admin'`. Non-admin redirected to `/`.

   g. **Member routes** (`pathname.includes('/member/')` and NOT `isReservationRoute(pathname)`):
      - First: check `getMembershipFromCookie(request)` for cached status
      - If cache hit and `cache.active === true`: skip DB query, proceed
      - If cache miss/expired: query `memberships` table for active status (same query as current)
      - If active: call `setMembershipCookie(supabaseResponse, { active: true, planType: membership.plan, cachedAt: Date.now() })`
      - If NOT active: redirect to `/pricing`
      - Note: also query `plan` column (not just `status`) so we can cache the plan type

   h. **Compose response**: Run `intlMiddleware(request)`, copy all Supabase cookies (and membership cache cookie) onto the intl response, return it.

5. **Keep the existing matcher** (broad for next-intl):
```typescript
export const config = {
  matcher: ['/((?!api|_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)'],
}
```

**Critical rules:**
- NEVER use `getSession()` — always `getUser()` (security requirement from codebase conventions)
- The membership query must select `status, plan` (not just `status`) for the cookie cache
- Reservation routes (/reservations, /checkout-session) skip membership gate but still require auth
- `/dashboard` without `/member/` prefix must also be protected (it's in PROTECTED_PREFIXES)
