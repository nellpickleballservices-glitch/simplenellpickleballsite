**app/actions/billing.ts** — Two Server Actions following the auth.ts pattern:

1. `createCheckoutSessionAction(planType: 'vip' | 'basic')`:
   - Get user via `createClient()` + `getUser()` — redirect to /login if no user
   - Resolve price ID from env vars: `STRIPE_PRICE_ID_VIP` or `STRIPE_PRICE_ID_BASIC`
   - Get origin from `headers()`
   - Create Stripe Checkout Session with: `mode: 'subscription'`, `line_items: [{ price: priceId, quantity: 1 }]`, `client_reference_id: user.id` (CRITICAL — this is how webhook maps to Supabase user), `customer_email: user.email`, `success_url: ${origin}/{locale}/checkout-success?session_id={CHECKOUT_SESSION_ID}`, `cancel_url: ${origin}/{locale}/pricing?cancelled=true`, `subscription_data.metadata: { supabase_user_id: user.id, plan_type: planType }`
   - NOTE: Use the user's current locale from the URL or default to 'es' for success/cancel URLs
   - `redirect(session.url!)`

2. `createPortalSessionAction()`:
   - Get user via `createClient()` + `getUser()` — redirect to /login if no user
   - Fetch `stripe_customer_id` from `memberships` table where `user_id = user.id`
   - If no membership found, throw error
   - Create portal session with `customer: membership.stripe_customer_id`, `return_url: ${origin}/{locale}/dashboard`
   - `redirect(portalSession.url)`
