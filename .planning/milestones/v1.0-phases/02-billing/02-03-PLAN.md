**app/[locale]/(member)/checkout-success/CheckoutSuccessClient.tsx** — Client Component ('use client'):
- State machine with 3 states: 'pending' | 'active' | 'timeout'
- On mount:
  1. Get current user via `supabase.auth.getUser()`
  2. First, check if membership is already active (webhook may have fired before redirect): query `memberships` where `user_id = user.id` and `status = 'active'`. If found, skip Realtime and go straight to 'active' state.
  3. Set 30-second timeout that transitions to 'timeout' state
  4. Subscribe to Supabase Realtime channel 'membership-status':
     ```typescript
     supabase.channel('membership-status')
       .on('postgres_changes', {
         event: '*',  // Listen for INSERT and UPDATE — first checkout creates a row
         schema: 'public',
         table: 'memberships',
         filter: `user_id=eq.${user.id}`,
       }, (payload) => {
         if (payload.new.status === 'active') {
           clearTimeout(timeout)
           setStatus('active')
           supabase.removeChannel(channel)
         }
       })
       .subscribe()
     ```
  5. Cleanup: remove channel and clear timeout on unmount
- **Pending state**: Animated "Processing your membership..." with loading spinner/pulse animation. Use NELL brand colors (lime pulse on midnight bg). Exciting energy — this is a celebration moment building.
- **Active state**: Success celebration! Per CONTEXT.md: "should match brand energy: Electric Lime, Caribbean vibrancy". Show confetti or burst animation (CSS-only, no library), congratulations message, plan name, and two CTAs: "Reserve a Court" (disabled/coming soon badge since Phase 3) and "Go to Dashboard".
- **Timeout state**: Reassurance message per CONTEXT.md: "This is taking a bit longer than usual. Your payment was received -- your membership will activate shortly." + "Check status" button that re-queries memberships and transitions to 'active' if found, or stays on timeout.
