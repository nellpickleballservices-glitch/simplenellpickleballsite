  <name>Task 1: Webhook handler functions module</name>
  <files>lib/stripe/webhookHandlers.ts</files>
  <action>
Create `lib/stripe/webhookHandlers.ts` with 5 exported handler functions. Each function takes the Stripe event object and `supabaseAdmin` client as params. Import `stripe` from `@/lib/stripe` for subscription retrieval.

**handleCheckoutCompleted(session: Stripe.Checkout.Session, supabase: SupabaseClient)**:
- Extract `client_reference_id` as `userId` — throw Error if missing ("Missing client_reference_id")
- Extract `subscription` (string) and `customer` (string) from session
- Call `stripe.subscriptions.retrieve(subscriptionId)` to get price and period
- Determine `planType` by comparing `subscription.items.data[0].price.id` to `process.env.STRIPE_PRICE_ID_VIP` — if match then 'vip', else 'basic'
- Calculate `periodEnd` from `subscription.current_period_end * 1000` as ISO string
- Upsert into `memberships`: `{ user_id: userId, plan_type: planType, status: 'active', payment_method: 'stripe', stripe_subscription_id: subscriptionId, stripe_customer_id: customerId, current_period_end: periodEnd, updated_at: new Date().toISOString() }` with `onConflict: 'user_id'`

**handleSubscriptionUpdated(subscription: Stripe.Subscription, supabase: SupabaseClient)**:
- Extract `customer` as string
- Determine `planType` from `subscription.items.data[0].price.id`
- Map Stripe status to app status: active->active, past_due->past_due, canceled->cancelled, unpaid->past_due, trialing->active. Default to 'past_due' for unknown statuses.
- Calculate `periodEnd` from `current_period_end`
- Update `memberships` where `stripe_customer_id = customerId`: `{ plan_type, status, current_period_end, updated_at }`

**handleSubscriptionDeleted(subscription: Stripe.Subscription, supabase: SupabaseClient)**:
- Update `memberships` where `stripe_subscription_id = subscription.id`: `{ status: 'cancelled', updated_at }`

**handleInvoicePaymentSucceeded(invoice: Stripe.Invoice, supabase: SupabaseClient)**:
- Extract `subscription` as string from invoice
- If no subscription ID, return early (one-time invoices don't apply)
- Retrieve subscription via `stripe.subscriptions.retrieve(subscriptionId)` to get `current_period_end`
- Update `memberships` where `stripe_subscription_id = subscriptionId`: `{ current_period_end: periodEnd, status: 'active', updated_at }`

**handleInvoicePaymentFailed(invoice: Stripe.Invoice, supabase: SupabaseClient)**:
- Extract `subscription` as string from invoice
- If no subscription ID, return early
- Update `memberships` where `stripe_subscription_id = subscriptionId`: `{ status: 'past_due', updated_at }`

IMPORTANT: All handlers use `supabaseAdmin` (service_role) — this is a server-to-server call with no user auth context. Regular Supabase clients would be blocked by RLS.

<task type="auto">
  <name>Task 2: Webhook Route Handler with signature verification and idempotency</name>
  <files>app/api/stripe/webhook/route.ts</files>
  <action>
Create `app/api/stripe/webhook/route.ts` — a Next.js App Router Route Handler exporting `POST`.

**Step 1: Raw body extraction**
- CRITICAL: Use `const body = await request.text()` — NOT `request.json()`. Using `.json()` re-serializes the body and corrupts the byte sequence needed for Stripe signature verification (see RESEARCH.md Pitfall 1).

**Step 2: Signature verification**
```typescript
const signature = request.headers.get('stripe-signature')
if (!signature) return new Response('Missing stripe-signature header', { status: 400 })

let event: Stripe.Event
try {
  event = stripe.webhooks.constructEvent(body, signature, process.env.STRIPE_WEBHOOK_SECRET!)
} catch (err) {
  return new Response('Webhook signature verification failed', { status: 400 })
}
```

**Step 3: Idempotency guard**
- Insert `{ stripe_event_id: event.id, event_type: event.type }` into `webhook_events` table using `supabaseAdmin`
- If error code is '23505' (unique_violation): return `Response('Duplicate event', { status: 200 })` — return 200 so Stripe stops retrying
- If other DB error: return `Response('DB error', { status: 500 })`

**Step 4: Event dispatch**
- Switch on `event.type` and call the appropriate handler from `lib/stripe/webhookHandlers.ts`:
  - 'checkout.session.completed' -> handleCheckoutCompleted(event.data.object as Stripe.Checkout.Session, supabaseAdmin)
  - 'customer.subscription.updated' -> handleSubscriptionUpdated(event.data.object as Stripe.Subscription, supabaseAdmin)
  - 'customer.subscription.deleted' -> handleSubscriptionDeleted(event.data.object as Stripe.Subscription, supabaseAdmin)
  - 'invoice.payment_succeeded' -> handleInvoicePaymentSucceeded(event.data.object as Stripe.Invoice, supabaseAdmin)
  - 'invoice.payment_failed' -> handleInvoicePaymentFailed(event.data.object as Stripe.Invoice, supabaseAdmin)
- Wrap switch in try/catch — if handler throws, return `Response('Handler error', { status: 500 })` so Stripe retries
- After successful processing: return `Response('OK', { status: 200 })`

**Step 5: Disable body parsing**
- App Router does NOT auto-parse the body, so no extra configuration needed (unlike Pages Router which required `bodyParser: false`)

Import `stripe` from `@/lib/stripe`, `supabaseAdmin` from `@/lib/supabase/admin`, all handlers from `@/lib/stripe/webhookHandlers`.

NOTE: This route is at `app/api/stripe/webhook/route.ts` — outside `[locale]` because it is an API endpoint, not a page. No i18n middleware applies.
